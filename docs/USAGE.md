RestaurantOS — Usage Guide

Quick start (Windows)

1) Run the installer (recommended)
- Installer path (built by jpackage):
  .\build\jpackage\restaurant-oms-1.0.exe
- Double-click the EXE and follow the installer wizard. After install the app is available in Start Menu or the installation folder.

2) Run the bundled distribution (no install)
- Run the bundled start script (Windows):
  .\build\distributions\restaurant-oms\restaurant-oms\bin\restaurant-oms.bat
- This launches the packaged distribution which includes the JavaFX runtime jars.

3) Run the JAR (developer)
- Build first (recommended):
  .\gradlew.bat clean build
- Run the JAR with JavaFX module-path (Windows example using the distribution's JavaFX libs generated by the Gradle build):
  java --module-path ".\build\distributions\restaurant-oms\restaurant-oms\lib" --add-modules javafx.controls,javafx.fxml -jar .\build\libs\restaurant-oms.jar
- If you prefer to use a separate JavaFX SDK, replace the module-path with the path to your JavaFX SDK lib directory.

4) Common jpackage command (if you need to re-create the installer)
- Example (Windows, requires WiX):
  jpackage --type exe --name restaurant-oms --input .\build\libs --main-jar restaurant-oms.jar --main-class com.restaurantos.App --dest .\build\jpackage --module-path .\build\distributions\restaurant-oms\restaurant-oms\lib --add-modules javafx.controls,javafx.fxml --win-shortcut --win-menu --verbose

Main areas
- Sidebar: navigate between Orders, Tables, Kitchen, Reports, Settings.
- Tables: click a table card to view details and change status. Selected table is highlighted.
- Orders: create, view, change priority and status; select an order to edit items.
- Kitchen Queue: columns for New / Preparing / Ready and move orders between states.
- Reports: KPIs and stats.
- Settings: edit restaurant info and app preferences.

Basic workflows
- Create order: click + New Order in Orders, add items from the product panel.
- Change table status: open Tables, click a table card and use Change Status in the side panel.
- Pay bill: open Bills and use Pay actions for pending bills.

Troubleshooting (when UI does not appear)
- Run from a visible terminal to see errors: use the same java command as above in PowerShell — exceptions and stack traces will print to the console.
- Error: "Module javafx.controls not found": the module-path is incorrect or JavaFX jars are missing. Point module-path to a folder with JavaFX jars (e.g. the distribution lib folder above or your JavaFX SDK lib).
- Error: no window appears but process runs: run the java command from the terminal to capture logs; ensure you have a Java runtime that supports JavaFX and not running headless.
- If building installer, ensure WiX Toolset is installed and on PATH for Windows.

Where to find files
- Source: src\main\java and src\main\resources
- Stylesheet: src\main\resources\css\style.css
- Logo asset used: src\main\resources\images\logo-generated.png
